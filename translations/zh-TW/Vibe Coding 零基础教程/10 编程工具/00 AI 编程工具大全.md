## 一、為什麼要了解程式設計工具？

在傳統程式設計時代，工具的選擇其實沒那麼重要。無論你用 VS Code 還是 Sublime Text，寫出來的程式碼都是一樣的。

但在 Vibe Coding 時代，**選對工具，可能讓你的開發效率提升 10 倍！**

為什麼這麼說？

因為不同的 AI 程式設計工具：

1. 能力差距很大：有的工具只能生成簡單程式碼，有的能幫你做整個專案
2. 適用場景不同：有的適合做原型，有的適合做產品，有的適合學習
3. 成本差異明顯：有的完全免費，有的每月要幾百塊
4. 學習難度不同：有的上手就能用，有的需要一定基礎

選對了工具，你可以事半功倍；選錯了工具，可能會讓你覺得 「Vibe Coding 也不過如此」。



## 二、AI 程式設計工具的 3 大類型

在深入了解具體工具之前，我們先來看看 AI 程式設計工具的基本分類。根據使用方式和複雜度，我把它們分為 3 大類：

![](https://pic.yupi.icu/1/toolstype%E5%A4%A7.jpeg)



### 零程式碼平台

在瀏覽器裡打開就能用，不需要安裝任何軟體，不需要懂任何程式碼。適合完全零基礎的新手、想快速做出原型的同學。

代表工具：Bolt.new、Lovable、秒噠

優勢：上手快、所見即所得、自動部署

局限：功能相對簡單，複雜專案可能力不從心

![](https://pic.yupi.icu/1/image-20260106100544197.png)



### AI 程式碼編輯器

需要下載安裝，介面像傳統程式碼編輯器，但內建了強大的 AI 助手。適合有一定基礎、想深入學習 Vibe Coding、需要做複雜專案的人。

代表工具：Cursor、Windsurf、Antigravity、Augment Code

優勢：功能強大、靈活度高、適合大型專案

局限：需要一定學習成本，對新手不夠友好

![](https://pic.yupi.icu/1/image-20260106100559696.png)



### 命令列工具

在終端裡通過命令列和 AI 對話，適合有程式設計基礎的開發者、喜歡命令列的極客。

代表工具：Claude Code、Gemini CLI

優勢：效率極高、自動化程度強、成本可控

局限：需要一定技術基礎，新手不建議使用

![](https://pic.yupi.icu/1/image-20260106100958506.png)
