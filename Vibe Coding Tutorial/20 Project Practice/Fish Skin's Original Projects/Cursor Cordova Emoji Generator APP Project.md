## I. What is Cordova?

Apache Cordova is an open-source mobile application development framework that allows developers to use web technologies like HTML, CSS, and JavaScript to develop **cross-platform** mobile applications. By encapsulating web technologies in native containers, it enables developers to write code once and then run it on multiple platforms like Android, iOS, Windows, and more.

![](https://pic.yupi.icu/1/1749628169384-9f90fa60-e328-46c8-99eb-6f4f343b8aeb.png)

Cordova is mainly implemented through the following core components‚Äîinterested students can learn more:

![](https://pic.yupi.icu/1/image-20250612190256834.png)

In other words, to develop an APP, we just need to give our website files to Cordova, install plugins as needed, modify configurations, and then directly use its build tools to package the web application into a native APP application (like an APK file)! It involves almost no code writing or development.

Sounds simple, right? Anyone can do it? But to use Cordova to develop APPs, you must install the corresponding environment on your computer, such as Android and iOS. The difficulty of installing the environment can be said to be **very explosive**!

## II. Environment Preparation

### Installing Cordova

First we need to install Cordova. Cordova depends on Node.js and NPM frontend tools. Just download from the [Node.js official website](https://nodejs.org/zh-cn) and it will automatically install NPM.

![](https://pic.yupi.icu/1/1749628757935-a1f29765-c0e8-4c3a-9486-e113f3814e8d.png)

You can understand NPM as a small tool for quickly installing various software. After installation, open the terminal and execute the following command to install Cordova:

```bash
npm install -g cordova
```

Cordova supports packaging websites as Android and iOS mobile apps, as well as Electron desktop applications. Below, Fish Skin will take you through installing what I personally consider the most difficult‚Äîthe **Android environment**. Note that each of the following steps isn't actually difficult, but you must read carefully! Not paying attention to one detail might cause an error!

### Generating Code

There are 2 generation modes here:

1. Create a Cordova project first, then perform AI code generation within that project. Tell AI you want to create a Cordova APP compatible website, and directly let AI generate code compatible with the APP. The advantage of this approach is that the generated code **can use Cordova plugins to call native system capabilities**, such as calling the camera to take photos.

2. Use AI to generate a website project separately outside the Cordova project. AI won't care whether you want to convert the project to a Cordova APP, then move the generated website into the Cordova project. The advantage of this approach is that the generated website code is easier to run, and it's **suitable for scenarios where you already have an existing website project**.

### Web Browsing

You can directly double-click the generated HTML file `www/index.html` to see the effect. Of course, it's more recommended to add a platform through cordova commands and run.

First add the browser platform:

```shell

```

If you encounter an error when executing the command, you can directly ask AI. For example, Fish Skin encountered a lack of command execution permission error:

![](https://pic.yupi.icu/1/1748597149478-5d93aa00-a15d-41a7-b7b3-6011c9098420.png)

The solution is to execute the following command to modify PowerShell's execution policy:

```bash
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

After successfully adding the platform, you can enter the `cordova run` command to run the platform:

```bash
cordova run browser
```

### Common Errors

Packaging and running is where you're most likely to encounter errors. You might encounter many types of errors, such as missing plugins, missing files, inability to install dependencies, inability to run, etc. It's recommended to directly send the error message to AI and let it help you solve it.

![](https://pic.yupi.icu/1/1748599524081-e2121c56-11bc-4504-bd2d-6a5fff26f4fb.png)



#### 3. Command Execution Failure

Executing `cordova run` results in command execution failure error, possibly because `cmdline-tools` wasn't configured in the environment variable Path.

![](https://pic.yupi.icu/1/1749607603308-b1dac4af-1505-4e15-b2ab-25ffd9abf3a5.png)



#### 4. Gradle Installation Failure

Even though Gradle is already installed, Cordova will still install Gradle, and might fail to download due to network reasons:

![](https://pic.yupi.icu/1/1749469918640-07f5eca4-5651-4e35-91a5-222c43363004.png)

At this point, we can configure the environment variable `CORDOVA_ANDROID_GRADLE_DISTRIBUTION_URL` to specify downloading Gradle locally. Set the environment variable value to the path of the Gradle archive we downloaded ourselves.

![](https://pic.yupi.icu/1/1749471697598-c7dd5754-4e58-4b01-9787-5889a0f9df52.png)

If after modifying the configuration and executing the packaging command again you still get an error, it's recommended to delete the `platforms/android/.gradle` cache in the project and retry.
